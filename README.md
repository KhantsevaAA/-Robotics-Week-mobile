Программный модуль автономной навигации и манипуляции для мобильного робота на базе ROS2, разработанный командой **misis_robolab** на хакатоне [MISIS Robotics Week](https://misis.ru/events/6739/) (21 февраля 2026 г.) для заказчика — НИТУ МИСИС. Проект решает задачу автоматизации складской логистики: робот автономно обнаруживает, классифицирует и доставляет объекты в зачётные зоны на игровом поле.

## Описание задания

Хакатон представлял собой соревнование на поле 1750×1250 мм с разметкой ArUco-маркерами. Одновременно действовали два робота, стартовые позиции и расположение 16 объектов определялись случайно. Задача состояла из двух частей:

**1. Автономная навигация и локализация** — робот должен был перемещаться по полю, локализуясь с помощью бортовой камеры, лидара и внешней камеры на П-образной мачте высотой 1500 мм.

**2. Обнаружение и доставка объектов** — в реальном времени обнаруживать и классифицировать объекты (белые кубы с ArUco-маркерами, красные/синие кубы, цилиндр, фигурки пингвина, осьминога, кролика) и доставлять их в зачётные зоны с учётом бонусной системы стекирования (например, кролик на кубе ID 20 давал 16 очков вместо базовых 4).

## 🚀 Установка и запуск

### Зависимости

- Ubuntu 22.04
- [ROS2 Humble](https://docs.ros.org/en/humble/Installation.html)
- Python 3.10+
- OpenCV 4.x
- Ultralytics YOLOv8
- `cv_bridge`, `geometry_msgs`, `sensor_msgs`

```bash
# Установка Python-зависимостей
pip install ultralytics opencv-python-headless

# Установка ROS2-зависимостей
sudo apt install ros-humble-cv-bridge ros-humble-vision-msgs
```

### Клонирование и сборка

```bash
git clone https://github.com/KhantsevaAA/-Robotics-Week-mobile.git
cd -Robotics-Week-mobile.git/ros2_ws

# Установка зависимостей пакетов
rosdep install --from-paths src --ignore-src -r -y

# Сборка
colcon build

# Активация окружения
source install/setup.bash
```

### Запуск

```bash
# Запуск основного стека навигации
ros2 launch frob_controller main_launch.py

# Запуск модуля компьютерного зрения (в отдельном терминале)
ros2 run vision_node vision_node
```

> **Примечание:** перед запуском убедитесь, что камера подключена и доступна как `/dev/video0`, а параметры ArUco-маркера робота заданы в конфиг-файле.

## ⚙️ Функционал

- **Локализация по ArUco** — определение координат (x, y) и угла поворота (yaw) робота по ArUco-маркеру на крыше через внешнюю камеру; публикация в топик как `Pose2D`
- **Обнаружение объектов через YOLO** — дообученная модель YOLOv8 для обнаружения и классификации всех игровых объектов; координаты публикуются в отдельный ROS2-топик
- **Обучение YOLO** - для обучения использовался проект https://github.com/Maksanigilator/Roshack-yolodocker
- **Автономное движение** — управление приводами робота Frob на основе данных локализации и карты объектов
- **Стратегия сбора** — планирование маршрута с учётом бонусной системы: приоритизация высокоочковых комбинаций (стекирование объектов)
- **Работа в локальной системе координат** — без привязки к единицам СИ, все вычисления в пикселях изображения
